## 6. Wat expliciet nog NIET werkt

### 6.1 `doctor --apply --yes`

CLI accepteert flags, maar **schrijft geen bestanden**. Fix-actions worden alleen gerapporteerd:

- `render_cursor_snippet` → `.cursor/mcp.kater.json`
- `render_env_example` → `.env.kater.example`

**Ontbrekend:** `src/kater/autofix.py` + wiring in `cli.py`.

### 6.2 Externe MCP adapters

Alle externe sources in `profiles.py` zijn **catalogus**. Geen stdio/http/sse proxy naar GitHub, Exa, etc.

### 6.3 Chain execution

`chains.py` = metadata. Geen `kater chain run`, geen MCP chain-tools.

### 6.4 Utrecht adapter

`adapters/utrecht.py` checkt alleen env/repo path. Geen call naar `UTRECHT_MCP_URL` of `utrecht mcp serve`.

### 6.5 Docker runtime

Compose mount `./config:/app/config:ro` maar **`config/` map bestaat niet**.

### 6.6 Cursor end-to-end

Docs aanwezig; geen bevestigde Cursor → `http://127.0.0.1:9090/sse` smoke.

### 6.7 CI

Geen `.github/workflows/` in kater-dev-tools.

## 7. Aanbevolen volgorde next sessie

### Tranche 1 — Autofix (U3 afmaken)

**Doel:** `doctor --apply --yes` schrijft veilige Kater-owned artifacts.

1. Maak `src/kater/autofix.py`:
   - `render_cursor_snippet(url, profile)` → JSON snippet
   - `render_env_example(profiles)` → env vars uit `TOOL_SOURCES`
   - `apply_fix_actions(report, output_dir)` → schrijft bestanden
2. Wire in `cli.py`: na `--apply --yes` daadwerkelijk schrijven + JSON summary
3. Tests: apply schrijft files, idempotent, geen Cursor global edits

**Acceptatie:**

```bash
uv run kater doctor --profile ops --fix-plan --apply --yes --json
# → .cursor/mcp.kater.json + .env.kater.example bestaan
```

### Tranche 2 — Eerste echte adapter (U7 + proof of U4)

**Doel:** één externe MCP via Kater, profile-gated.

**Optie A (Utrecht, laag risico):**

- Proxy naar bestaande `utrecht mcp serve` via SSE/stdio
- Env: `UTRECHT_MCP_URL`, `UTRECHT_REPO_PATH`
- Profile: `utrecht`

**Optie B (GitHub, hoog nut):**

- Stdio spawn of HTTP bridge naar GitHub MCP
- Profile: `ops` / `code`
- Alleen read-tools in v0 (list PRs, get status)

**Acceptatie:** Cursor ziet via Kater meer dan 2 tools wanneer profile ≠ core.
